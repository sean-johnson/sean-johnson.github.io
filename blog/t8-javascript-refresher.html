Technical 8 | Date: 28/5/2016
What different 'data-types' are there, and what are they?

First off what is data? Essentially, data is the stored information of sequenced 'bits'. Bits can be sequenced in the form of a 1s and a 0s, or, the shades of black or white, or even as a weak or strong elecrtical charge. The key here is that bits can be any kind of two-valued things. In the case of computers, the sequencing of 1s and 0s is what creates these bits and the combination of these sequences is what 'data' is. Data is informaton.

The combination of 1s and 0s can create (probably) an infinite number of data. For example: The combination bit sequence '01000001' is the binary representation in computers for the letter 'a' where as '01001110' is the letter 'n'. To be able to understand and make use of the infinite combination of bits, it is organised in groups. 8bits, 16bits, 32bits and 64bits are all ways that bits are grouped and interpreted to mean something.

Data types are the catergorisation of certain types of data. In JavaScript there are data types such as numbers, strings, Booleans, objects, arrays, functions, and undefined values.

I like to think that these types are the fundamental building blocks of the javascript language as well as give conceptual rules on how to structure and write the language.

Data structures; Arrays.

An array is a collection of values, grouped within square brackets. Arrays can 'hold' values and these values can be refrenced by calling what numeric index the value is stored in. To access the first value in the array, the number 0 (not 1) is used. for example:

var cars = ["Saab", "Volvo", "BMW"];

console.log(cars[0]);
// -> "Saab"
