Responsive Design.

Responsive design in web development means creating and developing web-page layouts that adapt and change depending on the screen size it is viewed on. It's a very important element to understand as a developer because viewer are not restricted to one screen size; whether they are viewing it on a tablet, their laptop or mobile device, the contents being viewed needs to cater to these various viewing sizes.

What is 'Mobile-first' design?

Mobile first means that you design a website from the viewing perspective of a mobile device, first. Why? Because mobile screens are the smallest, it challenges the designer / developer to really think about the essential elements that should be on the web page, it helps to identify in the simplist format what is most important. When this is defined, then you can about adding extra features or links in the larger screen formats. Mobile first helps to keep you focused on the objective.

What are frameworks?

Frameworks exist to make a developers job easier. Essentially, frameworks are pre-written code that act as a 'base' when starting a project. Usually CSS frameworks such as 'bootstrap' or 'skeleton' are a collection of css tags that help the developer with things such as layout and grid structure, typography, colours, button aesthetics, it has a good basic ground work so that you can spend more time on coding and less time on specific aesthetic details etc.

Frameworks complexity ranges and that can give both advantages and disadvantages when coding or designing a site. Some advantages of frameworks is that it gives you a clean template and usually 'clean' code. Frameworks can also save you alot of time without having to do the 'basic' code everytime you start a project. Disadvantages however, is that aesthetically, some frameworks (which are popular) can create work that is mundane and uncreative. For example, one can learn to see website that have been developed in certain frameworks; Bootstrap is a good example, where the layout and style of bootstrap is noticable which can make the site seem un-original.

Wireframes.

Wireframes are basically draft copies of what you want your site to look like. It gives a breif overview of elements and sections on the page (usually blocked out by squares). Its used as a rough template to structure your website when you develop it in code.
